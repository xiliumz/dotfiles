# Naming convention in typescript project

## Variables

- **Arrays**: Use plural names (e.g., `users`, `cities`)
- **Non-arrays**: Use singular names (e.g., `user`, `city`)
- **Booleans**: Prefix with `is` (preferred) or `has` (e.g., `isActive`, `hasPermission`)

## Functions

- Start with a verb (e.g., `findCityById()`, `getUserData()`, `validateEmail()`)

## Types & Interfaces

- Always use **singular** names (e.g., `User`, `City`, `Product`)
- For arrays, use singular type with `[]` notation (e.g., `City[]` not `Cities`)
